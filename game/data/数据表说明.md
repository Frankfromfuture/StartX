# 卡牌数据表 startx_data.xlsx —— 唯一数据源

游戏启动时实时读取 `data/startx_data.xlsx`，**它是卡牌/配方/卡包的唯一权威来源**。
改完保存、重启游戏即生效；加一行就是加一张卡/配方/卡包，无需改代码。

> 旧的 `cards.json / recipes.json / packs.json` 已不再被游戏读取（仅作历史备份 +
> `tools/gen_xlsx.py` 的初始种子）。`balance / research / idea_pools` 仍走各自 JSON。

## 三个工作表

### 1. cards（卡牌）
| 列 | 含义 | 示例 |
|---|---|---|
| id | 唯一英文标识（代码引用用） | `sales_rep` |
| name | 显示名 | `销售专员` |
| type | `employee` / `resource` / `facility` / `tool` / `customer` / `product` / `cash` / `business_model` | `employee` |
| workTags | 工作标签，逗号分隔（仅员工） | `sales,any` |
| salary | 月薪（$） | `2` |
| capacity | 产能 | `3` |
| spaceCapacity | 办公空间容量；场上所有卡的该数值之和即空间上限 | `30` |
| value | 价值（$）；生产时分别只取堆叠中最下面一张 `tool` 和最下面一张 `product` 的价值作为产能 | `1` |
| maxUses | 可用次数（`resource`，留空=无限/不适用） | `3` |
| workRequired | 生产该卡需要的总工作量；`生产时间 = workRequired ÷ max(1, 堆叠总产能)` | `30` |
| cost | 一次性生产成本（仅 `product` 使用，开工时扣除） | `1` |
| recipeId / businessModelId / unlocksRecipeId | 可选：商业模式卡关联的配方 id；留空则不关联 | `p1_make_rawprod` |

### 2. recipes（配方）
| 列 | 含义 | 示例 |
|---|---|---|
| id | 唯一标识 | `gather_lead` |
| name | 显示名 | `线索开发` |
| requiredIdeaId | 解锁所需研发 id（留空=初始可用） | `idea_auto` |
| worker_tags | 可执行的员工标签，逗号分隔 | `sales,any` |
| duration | 公式计算的参考生产时间，游戏不读取；按配方明确列出的原始输入卡计算 | `20.00` |
| input1 / input1Count / input1Consume ... input5 / input5Count / input5Consume | 最多 5 个输入卡；每组为卡 id、需要数量、是否消耗 | `market_lead_pool` / `1` / `false` |
| output1 / output1Count ... output5 / output5Count | 最多 5 个产出卡；每组为卡 id、产出数量 | `lead` / `2` |
| output_zone | 产出所在区（留空=默认） | `market` |
| packId | 该配方商业模式所属卡包；商业模式只会从此卡包抽到 | `garage_pack` |

`consume=false` 表示输入卡不被消耗（资源可反复产出）；`true` 表示用掉。

### 3. packs（卡包）
| 列 | 含义 | 示例 |
|---|---|---|
| id | 唯一标识 | `startup` |
| name | 显示名 | `创业起步包` |
| stage | 解锁阶段（0=车库,1=验证,2=产品,3=获客,4=规模,5=融资,6=上市） | `0` |
| price | 价格（$） | `3` |
| minCards / maxCards | 单次开包出卡数量范围 | `3` / `4` |
| slot1Card1 / slot1Prob1 ... slot5Card5 / slot5Prob5 | 最多 5 个抽卡槽位，每个 slot 表示一次独立抽卡；每槽最多 5 个候选；每个候选为卡 id + 权重数字 | `sales_rep` / `40` |

**slot 编码**：同一槽位内的概率数字是相对权重。例：`slot1Card1=a, slot1Prob1=40, slot1Card2=b, slot1Prob2=60` 表示第一槽 40% 出 a、60% 出 b；`slot2Card1=c, slot2Prob1=100` 表示第二槽必出 c。

## 编辑要点
- **id 不要重复**；id 为空的行会被跳过（可用作空行/注释行）。
- 数值留空按 0 处理；`maxUses` 留空表示该卡没有次数限制。所有配方产出卡必须填写 `workRequired`。
- 生产时间由固定公式实时计算：`workRequired ÷ max(1, 堆叠总产能)`，配方表不设置单独的时长。
- `recipes.duration` 是方便策划查看的公式参考列，不参与游戏读取；修改卡牌工作量、产能或价值后会自动更新。
- 列顺序可调、可加注释列——解析按**表头名**取值，不认列位置。
- 新增卡/配方/卡包后，图鉴、配方书、开包逻辑都会自动纳入，无需改代码。

## 注意
- Godot **不能原生读 xlsx**，本项目内置了解析器（`scripts/XlsxReader.gd`，启动一次性解析，
  运行时零开销）。它按 OOXML 标准实现，兼容 Excel/Numbers 重新保存后的文件。
- **导出成品时**：xlsx 是非资源文件，需在导出预设里把 `data/*.xlsx` 加入「导出过滤」，
  否则打包后读不到（开发期从项目目录直接运行不受影响）。
- 想从当前 JSON 重新生成种子表：`python3 tools/gen_xlsx.py`。
