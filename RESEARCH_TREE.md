# StartX · 企业研发路线图（Research Tree / 科技树）

> 把 GDD 中"随机掉落的 Idea 卡"升级为 **RimWorld 式主动研发树**：
> 派员工常驻**研究牌**、随时间积累**研发点数 RP**、按分支主动选取解锁 **Idea**，Idea 解锁新**配方**。
> 与 [RECIPE_BOOK.md](RECIPE_BOOK.md) 的配方、[BOOSTER_PACKS.md](BOOSTER_PACKS.md) 的阶段一一对应。

---

## 0. 与原 GDD 的关系（改动点）

| 维度 | 原 GDD | 本方案 |
|------|--------|--------|
| Idea 获取 | 卡包里一次性随机掉落 | 研发树主动解锁，花 RP 点选分支 |
| 进度感 | 靠运气 | 确定性、可规划（像研发点数） |
| 卡包里的 Idea slot | 给 Idea 卡 | 改为给 **RP 加速道具 / 研究资料**（+一次性 RP）|
| 配方可见性 | 未解锁隐藏 | 同左：未解锁的 Idea 对应配方在 Recipe Book 显示为 🔒 |

核心张力：**"始终保持一个研究牌有人工作"**——研究要占用一名员工，早期把人拉去研究就少一个人做生产，玩家需在"扩张产能"与"解锁科技"间权衡。

## 1. 研发点数（RP）经济

### 1.1 研究牌（Research Bench）
- 棋盘上有一张**研究牌**（开局即有 T1）。把**员工**拖到研究牌上 → 该员工进入 `researching` 状态，按时间产出 RP。
- 研究牌有**席位数**（随硬件升级增加），可同时安排多名研究员叠加产出。
- 研究牌**硬件可升级**（消耗现金/资源），提升 RP 倍率与席位。

### 1.2 硬件等级表（研究牌升级）

| 等级 | 名称 | RP 倍率 | 席位 | 升级配方 |
|:---:|------|:---:|:---:|----------|
| T1 | 笔记本工位 | ×1.0 | 1 | 开局自带 |
| T2 | 工作站 | ×1.6 | 2 | 现金 ×4 + 工程师 |
| T3 | 研发实验室 | ×2.5 | 3 | 现金 ×8 + 高级工程师 + 设备采购 |
| T4 | AI 算力集群 | ×4.0 | 4 | 现金 ×15 + 数据湖 + AI Copilot |

### 1.3 研究员素质加成（不同员工产 RP 不同）

| 研究员 | 单位 RP 倍率 | 备注 |
|------|:---:|------|
| 普通员工 / 创始人 | ×1.0 | 兜底 |
| 研发专员 Engineer | ×1.2 | |
| 高级工程师 Senior Eng | ×1.5 | |
| 数据分析师 Data Analyst | ×1.3 | 有概率暴击（一次产出 ×2）|
| 外部顾问 Consultant | ×1.4 | 偶发额外产出"行业报告"(+一次性 RP) |

### 1.4 装备加成（搭配先进装备增加单位产 RP）

| 装备 | 效果 |
|------|------|
| AI Copilot | 该研究员 RP +50% |
| 双屏/外设 | RP +15% |
| 行业数据库订阅 | 全研究牌 RP +20%（设施级）|
| 论文/专利资料（消耗品）| 投入后一次性 +N RP |

### 1.5 产出公式

```
rpRate(每秒) = benchMult(硬件) × Σ_研究员[ baseRate × 员工倍率 × (1 + Σ装备加成) ]
baseRate = 0.2 RP/s  (≈ 单人基础 12 RP/分钟)
```

> 例：T2 工作站(×1.6) + 高级工程师(×1.5) + AI Copilot(+50%)
> = 1.6 × (0.2 × 1.5 × 1.5) = **0.72 RP/s ≈ 43 RP/分钟**。

### 1.6 RP 成本分层（节点定价）

| 层级 Tier | 典型 RP 成本 | 大致出现阶段 |
|:---:|:---:|------|
| T1 | 15–30 | 起步 / 产品 |
| T2 | 40–80 | 经营 |
| T3 | 100–170 | 资本 / 扩张 |
| T4 | 200–360 | 终局 |

## 2. 研发树结构（6 大分支）

```
                       ┌── A 产品技术线 ──┐
                       ├── B 销售增长线 ──┤
  [研究牌·RP] ────────▶├── C 组织中后台线 ┼──▶ 解锁 Idea ──▶ 解锁配方/设施/市场
                       ├── D 资本金融线 ──┤
                       ├── E 合规产业线 ──┤
                       └── F 自动化/全球/上市线（终局）┘
```

- 每个分支 3–4 层，节点有 **RP 成本 + 前置节点**。
- 节点解锁后永久生效；对应配方从 🔒 变为可用。
- 部分高阶节点需要**跨分支前置**（如 IPO 需要资本线 + 合规线）。

## 4. 与阶段 / 胜利路线结合

| 阶段（卡包） | 建议优先研发节点 | 对应胜利路线 |
|------|------|------|
| 起步 | C1 招聘 | — |
| 产品 | A1 MVP → B1 顾问销售 → A2 质量 | A 商业化打底 |
| 经营 | B2 签约 → C3 制度 → E1 合规 → B3 续约 | A 商业化、稳现金流 |
| 资本 | D1 财报 → D2/D3/D4 融资链 | **B 融资扩张** |
| 扩张 | C4 风控 → F1 自动化 → E2/E3 产业许可 | **C 产业准入** |
| 终局 | F2 海外 / F3 上市 | **D 上市**、**E 市场领导者** |

> 不同胜利路线 = 不同研发主线侧重，鼓励多周目走不同分支（重玩性）。

## 5. 数据模型（实现参考）

```ts
type IdeaNode = {
  id: string;            // 'idea_mvp'
  name: string;          // '精益 MVP'
  branch: 'A'|'B'|'C'|'D'|'E'|'F';
  tier: 1|2|3|4;
  stage: 'start'|'product'|'ops'|'capital'|'expand'|'endgame';
  rpCost: number;
  prereq: string[];      // 前置 IdeaNode id
  unlocksRecipes: string[];   // RECIPE_BOOK 的 recipe id
  unlocksFacilities?: string[];
  unlocksMarkets?: string[];
  oneTime: true;
};

type ResearchState = {
  rp: number;
  rpRate: number;        // 每秒，由研究牌实时计算
  unlocked: Set<string>; // 已解锁 IdeaNode id
  benchTier: 1|2|3|4;
  seatsUsed: number;
};
```

**配方门控**：`RecipeDefinition.requiredIdeaId` 不在 `unlocked` 中则该配方不可触发、Recipe Book 显示 🔒（GDD 4.2 已有此字段）。

## 6. 与当前 MVP 切片的衔接（后续实现步骤）

当前切片：所有配方可用 + "首次合成给发现奖励"。改造为研发树：

1. **新增 autoload `Research.gd`**：持有 `rp / rpRate / unlocked / benchTier`，每帧累加 RP。
2. **研究牌卡 + 席位**：`Board.gd` 识别员工进入研究牌栈 → 计入 `rpRate`（用 §1.5 公式）。
3. **HUD 顶栏加 RP 条**与 `🔬 研究中` 指示。
4. **研发树面板**（新 UI）：点节点花 RP 解锁，置灰前置未满足/RP 不足的节点。
5. **配方门控**：`recipes.json` 增 `requiredIdeaId`；`RecipeResolver` 校验 `Research.unlocked`。
6. **卡包改造**：`packs.json` 的 Idea slot 换成"研究资料"（一次性 +RP）。
7. 把"首次发现闪光"保留为**解锁节点时**的庆祝果汁。

数据建议落在 `game/data/research.json`（节点定义）。
